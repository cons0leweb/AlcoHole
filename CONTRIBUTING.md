# Contributing to AlcoHole

Thanks for your interest in contributing!

## Getting Started

1. Fork the repository and clone it locally.
2. Create a branch for your feature or fix:
   ```bash
   git checkout -b feature/my-feature
   ```
3. Make your changes and test them on a Paper/Spigot server.
4. Commit with a clear message:
   ```bash
   git commit -m "Add: description of change"
   ```

## Code Style

- Follow standard Java conventions (camelCase, PascalCase for classes).
- Use 4 spaces for indentation.
- Keep methods short and focused.
- Add comments for non-obvious logic.

## Commit Messages

Use the following prefixes:
- `Add:` for new features
- `Fix:` for bug fixes
- `Docs:` for documentation changes
- `Refactor:` for code restructuring
- `Chore:` for build/tooling changes

## Pull Requests

1. Ensure your PR targets the `main` branch.
2. Describe what your change does and why.
3. Link any related issues.
4. Make sure CI passes (build + tests).

## Reporting Issues

- Use the GitHub Issues tab.
- Include your server version, plugin version, and steps to reproduce.
- Attach relevant logs or screenshots.
